Definitions I learned today
Commit: taking a snapshot of your changes.
Main: the official version of the code lives here.
Branch: a copied version of the main branch where you can make edits without disturbing the main version.
Merge: combining your branch back into main when you are confident with your changes.
Pull request: a proposal to merge a branch into main, where others review the changes before they are published.
Flow of doing things on GitHub
Git clone > git checkout > git add > git commit > git push > pull request on GitHub.com.

Tool relationship
Homebrew installs system tools, including FNM. FNM installs Node and lets you switch versions. Node comes with npm, and npm installs JavaScript libraries inside specific projects.

PR — pull request.
repo — repository (your project folder, with its history).
commit — already defined.
diff — difference between two versions.
prod — production (the live, real-world version of an app).
dev — development (your work-in-progress version).
CLI — command-line interface (the terminal).
IDE — integrated development environment (e.g., VS Code).
CI/CD — continuous integration / continuous deployment (we'll cover in Phase 4; you can ignore for now).
